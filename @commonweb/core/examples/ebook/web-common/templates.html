Template
Template es la clase abstracta principal que sirve como base para el resto de plantillas en la biblioteca. Su función principal es resolver las directivas principales del framework dentro del árbol de elementos hijos.

Existen dos tipos de Templates pricipales que veremos a continuacion que son StaticTemplate y LazyTemplate sus nombres son bastante descriptivos en cuanto a su principal y quizas unica diferencia.


Directivas
Nombre Descripcion
for-each Renderiza un template definido por cada uno de los elementos en el contexto dato
show-if Evalua la condicion para ocultar/mostrar el elemento segun el resultado

Atributos
Nombre Descripcion
context Contexto de datos que seran usados en el template y propagados en la interpolacion
Static Template
Es un CustomElement que extiende de la clase Template heredando todas las funcionalidades de esta, agregando tambien un contexto de informacion al arbol de elementos, permitiendo la interpolacion de valores tanto atributos como de texto dentro de las vistas. 

<static-template context='{"name":"World"}'> 
  <h4>Hello {@host.data.name}</h4>
</static-template>

El resultado del previo snipped seria el siguiente : 

<static-template context='{"name":"World"}'> 
  <h4>Hello World</h4>
</static-template>

como podemos ver el valor de {@host.data.name} ha sido remplazado por World algo muy comun en las actuales librerias/frameworks web como Angular o React, para mas detalle ver la seccion Template Interpolation
